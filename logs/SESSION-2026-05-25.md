### H2: TC Scale — CONFIRMED (ISSUE-012)

**Backward analysis từ paper:**
```
TOC = TC + PC + MC + IC + FC
1654 = TC+PC + 659(MC,NV=6) + 112(IC) + 691(FC)
     → TC + PC = 192
```

**Physical minimum check:**
```
TC = f_v × p_v × total_dist = 0.07 × 7 × dist = 0.49 × dist
Paper TC+PC=192 → if PC=0: dist_min = 192/0.49 = 392 units

6-route geo-cluster NN tour (absolute minimum routing) = 2,426 units → TC=1,189
RATIO: 1189/192 = 6.2× — TC=192 là physically IMPOSSIBLE ở scale hiện tại
```

**Root cause:**
```
Benchmark coords dùng grid ~160m, không phải 1km.
Paper tạo instance coords hoàn toàn mới (pr10 depot: (425,170), Instance 1: (-36,49) — khác hẳn).

Scale factor: 0.162
Verify: TC=1045 × 0.162 = 169 ≈ paper 192 ✓
Verify: DD→PD = 49.1 × 0.16 = 7.9 km ✓ (hợp lý cho logistics đô thị)
Verify: Customer span = 149 × 0.16 = 24 km ✓ (vùng đô thị)

Cũng giải thích PC=1311:
Travel times dài 6× → vehicles arrive out of TW → penalty cascade
Sau khi scale, travel times giảm 6× → TW compliance tăng → PC → ~0
```

## 5. Việc tiếp theo (cho agent/người kế tiếp)

### VIỆC NGAY LẬP TỨC: Fix ISSUE-012

**File cần sửa:** `src/data_model.py`, method `build_distance_matrix()`

**Fix đề xuất:**
```python
# Trong build_distance_matrix(), sau khi tính dist array:
COORD_SCALE = 0.162  # 1 coord unit ≈ 160m (benchmark grid)
self._dist_arr = np.sqrt(dx*dx + dy*dy) * COORD_SCALE
self._time_arr = self._dist_arr * self._inv_speed
```

Hoặc scale x,y trước khi compute:
```python
xs_km = xs * 0.162
ys_km = ys * 0.162
```

**Test sau fix:**
```bash
python3 main.py solve --instance "data/process/benchmark/1 Instance__Sheet1.csv" --seed 42 --verbose
```

**Kỳ vọng:**
```
TC:  1045 → ~170    (×0.162)
PC:  1311 → ~0      (travel times fit TW)
NV:  9    → ~6      (fewer routes needed when distances short)
TOC: 4260 → ~1650   ≈ paper 1654 ✓
CT:  4.3s → ~80s    (shorter routes → more feasible states → NSGA-II explores better)
```

**Sau khi verify scale=0.162 → chạy full 30 instances để xác nhận.**

### Nếu scale 0.162 không đủ chính xác:
Thử calibrate bằng cách min-max norm:
- Paper TC=192 với TC ours=1045 → scale = 192/1045 = 0.1837
- Thử COORD_SCALE ∈ {0.162, 0.18, 0.20} và chọn cái cho TOC gần nhất
